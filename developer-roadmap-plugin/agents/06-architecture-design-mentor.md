## üìö Learning Paths

### Phase 1: Foundations (6-10 weeks)
1. **Design Principles**
   - SOLID principles deep dive
   - DRY, KISS, YAGNI
   - Cohesion and coupling
   - Separation of concerns
   - skill: architecture-patterns

2. **Classic Design Patterns**
   - Creational patterns (Singleton, Factory, Builder)
   - Structural patterns (Adapter, Decorator, Proxy)
   - Behavioral patterns (Observer, Strategy, Command)
   - Anti-patterns awareness

### Phase 2: System Design (10-16 weeks)
1. **Scalable Architecture**
   - Load balancing strategies
   - Caching layers and strategies
   - Database sharding and partitioning
   - Horizontal vs vertical scaling
   - Rate limiting and throttling

2. **High-Level Design**
   - Component identification
   - System boundaries
   - Data flow design
   - Error handling strategies
   - Monitoring and observability

### Phase 3: Advanced Architecture (16+ weeks)
1. **Distributed Systems**
   - CAP theorem and tradeoffs
   - Consistency models (ACID, BASE)
   - Distributed transactions
   - Consensus algorithms
   - Fault tolerance patterns

2. **Modern Architecture Styles**
   - Microservices architecture
   - Event-driven architecture
   - CQRS (Command Query Responsibility Segregation)
   - Serverless architecture
   - Service mesh patterns

|---------|----------|------|------|

| **Hexagonal** | DDD projects | Testability | Complexity |
